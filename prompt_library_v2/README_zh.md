# Prompt Library V2（提示词库 V2）

[English](README.md) | **中文** · [← 返回主 README](../README_zh.md)

一个基于 SQLite 的分层提示词库。你在浮动窗口里搭建「文件夹 + 条目」的树，然后在节点模板里引用它们，节点在执行时解析。

## 节点

两个节点（正向 / 负向变体——极性只决定文件夹树显示哪些条目）：

- **XYZ Prompt Library V2 Positive**
- **XYZ Prompt Library V2 Negative**

| 端口 | 类型 | 说明 |
|---|---|---|
| `prompt_template`（输入） | STRING（多行） | 使用下方语法的模板文本 |
| `seed`（输入） | INT | 驱动条目的随机模式（select / dropout / shuffle） |
| `resolved_prompt`（输出） | STRING | 完全解析后的文本 |
| `raw_template`（输出） | STRING | 未改动的原始模板（便于串联/调试） |

每次运行都会重新解析（库数据在两次运行之间可能已变化）。

## 打开窗口

- 每个节点上都有 **Library**、**Editor**、**Preview** 按钮。
- 或用顶栏 **XYZ Tools** 菜单 → *Prompt Library V2 — Library* / *— Text Editor*。

## 模板语法

### `[ref]` —— 条目引用

插入另一个条目的解析文本：

```
[character_name]            ← 用触发名（最短唯一别名）或完整路径
[character_name.expression] ← 用点号路径进入子条目
```

一个引用只会解析到唯一一个条目：完整路径优先于触发别名；多段引用取最长匹配前缀，其余作为子路径。未知引用解析为空。循环引用会被检测并跳过。

### `(text:1.2)` —— 权重

权重 ≠ 1.0 的提示词会被包裹输出，例如 `(glowing eyes:1.3)`。

### `{a|b|c}` —— 选项

> **当前行为：** 节点对 `{a|b|c}` 永远取**第一个**选项（`a`）。按批次选择的逻辑没有接进节点。需要变化时，请改用条目的**随机模式**（见下），由节点的 `seed` 驱动。

### 自动清理

解析后，引擎会合并连续的分隔符（例如空 `[ref]` 留下的）、删掉行首或整段末尾孤立的分隔符、并压缩重复空格。**换行会被保留**，作为你的段落结构。

## 库窗口（Library）

### 文件夹树

- 新建 / 重命名 / 移动 / 删除 文件夹与条目（右键菜单），拖拽排序。
- 独立的**极性**筛选（正向 / 负向 / 全部）和**使用中**筛选。
- 一键全部折叠 / 展开。

### 条目详情

- **名称**和**正向 / 负向**徽章。
- **触发别名** —— 可在 `[ref]` 里使用的短名。用 **+ alias** 自行添加；每个条目还有一个自动触发名（其路径的最短唯一后缀）。
- **提示词列表** —— 竖排或紧凑布局。可逐条启用/停用、设权重、拖拽排序。启用的在前、按提示词文本顺序排列；停用的在后、按字母顺序排列。
- **Mode** —— `off`、`select`（在 min/max 间随机取个数）、`dropout`（按概率丢弃每条），外加一个 **Shuffle** 开关。全部由节点 `seed` 驱动。
- **Format** —— 包裹每条提示词，例如 `art by {prompt}`（`{prompt}` 和 `{p}` 是占位符）。
- **Delimiter** —— 连接本条目各提示词的分隔符（`, `、` | `、换行……）。
- **子条目** —— 子条目（自有 / 继承自 `_template` / 覆盖）。插入父条目时，`_neg` 子条目可自动插入负向节点。每个子条目行有：**⤴** 把引用加入*本*条目的提示词、**＋** 把引用插入文本编辑器、**→** 打开它。

## 文本编辑器窗口（Text Editor）

- **单栏 / 分栏** —— 用可拖动的分隔条同时编辑正向和负向。
- 从节点的 **📝 Editor** 按钮打开编辑器会聚焦该节点：单栏模式下自动切换到该节点的正向/负向标签并载入它；分栏模式下只聚焦对应的窗格（不切换标签）。
- `[ref]` **高亮** —— 有效引用会着色；背景层跟随文本。
- 每栏独立的**撤销 / 重做**、**查找 / 替换**（区分大小写 / 全词 / 选区），以及按光标上下文添加分隔符的**智能插入**。
- **右键** —— 把选中文本加入某条目、由它新建条目、或打开被引用的条目详情。

### 编辑器里的自动补全

- 输入 `[` 触发条目/触发名建议；选中后插入 `[name]`（一个引用）。
- 输入 `/name` 则插入该条目的**解析文本**，而不是引用。
- （开关在 **XYZ Prompt Tools → Library**。）

## 插入 / 规范化

在 **XYZ Prompt Tools → Insertion** 配置（同时作用于库和标签自动补全）：

| 设置 | 效果 |
|---|---|
| 下划线替换为空格 | 插入 `blue eyes` 而非 `blue_eyes` |
| 自动逗号 | 插入标签后追加 `, ` |
| 转义括号 / 反斜杠 | 把非权重的 `()` 变成 `\(\)`，使其为字面量 |
| 全角 → 半角 | 把 `（）`、`，` 等转成 ASCII |
| 逗号空格 | 把逗号 + 任意个空格规范化为单个 `, `（不影响换行） |

## 命名规则

- **条目 / 文件夹名**不能包含：`.` `,` `|` `/` `\` `[` `]`
- **触发名**不能包含空格或 `,` `|` `/` `\` `[` `]`（允许 `.`——它作为路径分隔符）
